# Patent Specification

## Title of Invention
Data Sovereignty Learning Refusal System and Model Contamination Prevention Method

## Technical Field
**[0001]**
The present invention relates to data privacy protection and copyright management technology, and particularly to a Data Sovereignty Learning Refusal System and Model Contamination Prevention Method where data owners technically refuse or impede unauthorized learning (scraping) by AI models, or retroactively invalidate learning results.

## Background Art
**[0002]**
In current AI development, the practice of "freely learning" publicly available data on the Internet prevails, and data is used to improve AI capabilities regardless of the intentions of data creators (creators, individuals).

Existing refusal means such as "robots.txt" assume that AI crawlers will courteously comply, and have no enforcement power. Also, removing the influence (memory) of once-learned data from within a massive model (Unlearning) has been considered technically difficult.

## Summary of Invention
### Problem to be Solved
**[0003]**
The present invention aims to establish substantive learning refusal rights and data sovereignty by providing data owners with active defense means that physically and mathematically turn AI learning into "poison," making AI models that perform unauthorized learning bear the risk of performance degradation.

### Means for Solving the Problem
**[0004]**
The system of the present invention comprises the following configuration:

1. **Data Processing Unit** that adds adversarial perturbations to public data (images, text, audio) that are imperceptible to humans but maximize or diverge the error function in AI's gradient descent learning process.

2. **Learning Determination Unit** (application of membership inference attacks) that inputs data containing specific triggers (watermarks, etc.) to AI model inference APIs and detects from the output response whether that model has learned the owner's data without permission.

3. **Rights Exercise Unit** that, when unauthorized learning is detected, activates "forgetting triggers" contained in data to destroy specific weights in the model, or automatically issues legal deletion requests (GDPR, etc.).

## Effects of Invention
**[0005]**
According to the present invention, AI development companies will bear the risk that "the more data they collect without permission, the more their model performance degrades." This creates a situation where formal permission and compensation payment from data owners becomes economically essential for learning high-quality data, enabling formation of a healthy data distribution market.

# Claims

**Claim 1**
A learning refusal system for data owners to control machine learning of their data by artificial intelligence, comprising:
processing means that embeds, within a range not affecting human perception, adversarial noise patterns that increase loss for machine learning model learning algorithms (gradient descent, etc.) or impede specific feature extraction, into target digital data (images, text, audio, etc.); and
distribution means that publishes or distributes protected data generated by the processing means;
characterized by degrading inference accuracy or generalization performance of machine learning models that learn the protected data without permission.

**Claim 2**
The Learning Refusal System according to Claim 1, wherein:
the adversarial noise patterns include backdoor-type triggers designed to guide machine learning model outputs to specific error classes, or render outputs meaningless, only when specific trigger inputs are given.

**Claim 3**
The Learning Refusal System according to Claim 1 or 2, further comprising:
learning detection means that transmits probe inputs (test data) related to the protected data to machine learning model APIs on networks, and analyzes response results to calculate the probability (membership probability) that the machine learning model has incorporated the protected data into learning.

**Claim 4**
The Learning Refusal System according to any one of Claims 1 to 3, wherein:
the protected data has metadata including data owner signatures and learning permission conditions (royalties, periods, etc.) embedded as digital watermarks; and
in legitimate permitted learning processes, decryption keys are provided that cancel or remove effects of the adversarial noise patterns.

**Claim 5**
The Learning Refusal System according to any one of Claims 1 to 4, further comprising:
rights exercise proxy means that, when unauthorized learning is detected by the learning detection means, automatically sends deletion requests, usage fee claims, or legal notices (Cease and Desist) to operators of the machine learning model.

## Detailed Description of Embodiments
**[0006]**
Hereinafter, embodiments of the present invention will be described in detail.

### Embodiment 1: Embedding Adversarial Noise (Poison)
When uploading images to illustration SNS or blogs, they are converted to "protected images" through this system.

**1. Noise Generation:**
For original image I, calculate minute perturbation δ such that general image recognition models like CNN misrecognize "cat" as "toaster" (using Projected Gradient Descent method, etc.).
```
I' = I + δ
```
At this time, the L2 norm of δ is limited so humans cannot distinguish I from I'.

**2. Effect:**
When AI crawlers learn large amounts of this I', the model's feature space becomes distorted, and it can no longer correctly recognize even normal "cat" images (model contamination).
This forces development companies to use clean (permitted) datasets because "indiscriminately collecting images from the net makes the model stupid."

### Embodiment 2: Monitoring via Membership Inference
Monitor whether your data is being used without permission.

**1. Probe Generation:**
Protected data contains specific unique patterns (watermarks).

**2. API Testing:**
Input prompts or image fragments related to that watermark to target AI models (e.g., image generation AI).
If the model overreacts (Overfitting) to that watermark pattern, or generates images containing the watermark, it can be determined that "it was included in training data."

**3. Automatic Rights Exercise:**
Upon detection, automatically send DMCA applications or invoices by email along with evidence logs on blockchain.

### Embodiment 3: Path to Legitimate Licensing
This system supports not only "absolute learning prohibition" but also "learning OK if you pay."
Data purchasers are provided with "removal filters (Keys)" that cancel noise embedded in images.
By applying this as preprocessing, AI can learn clean images.
This allows data owners to leverage "learning refusal rights" to establish a business model of "data sales."

## Examples
**[0008]**
The following shows a specific example of the present invention.

As Example 1, a case of illustrator rights protection and countermeasures against unauthorized learning models is described.
The target is popular Illustrator A with a specific drawing style. Illustrator A is concerned about their works being used in image generation AI training without permission and being imitated.

In this example, protection was implemented through the following procedure:

**1. Data Processing:** When publishing new illustrations, Illustrator A applied this system's "style protection filter." This filter adds minute noise (adversarial perturbation) invisible to human eyes, designed so that when AI tries to learn this image, "style feature vectors" scatter in random directions during gradient calculation.

**2. Distribution:** Processed images were published on SNS.

**3. Unauthorized Learning:** AI development Company B's crawler collected these images without permission and incorporated them into the next model's training set.

**4. Model Contamination:** As Company B's model proceeded with learning, the loss function did not converge but instead began to diverge. Generated images, far from imitating Illustrator A's style, only output distorted, noise-mixed images (poisoning effect).

**5. Rights Exercise:** Simultaneously, digital watermarks (learning detection triggers) contained in images reacted, and membership inference detected the fact that "learned by Company B's model." The system automatically sent "copyright infringement notice and data deletion request" to Company B.

As a result, Company B was forced to exclude Illustrator A's data from the training set to prevent model performance degradation. Subsequently, Company B applied to Illustrator A for a legitimate license contract (including provision of poisoning removal keys), and appropriate compensation was paid. This established substantive learning refusal rights based on technical enforcement.

The present invention is applicable as rights protection tools for creators (illustrators, writers), corporate confidential information leakage prevention systems, personal information AI learning prevention privacy tools, etc., contributing to formation of a healthy AI data market.

# Abstract

## Problem
Public data on the Internet is learned without permission and used for AI capability improvement contrary to data creator intentions, while existing refusal means have no enforcement power.

## Solution
Processing means that embeds adversarial noise patterns that increase loss for machine learning model learning algorithms into digital data, and distribution means that publishes or distributes the protected data, degrading inference accuracy or generalization performance of machine learning models that learn the protected data without permission.

## Executive Summary
現在のAI安全対策（RLHFやConstitutional AI）は「ソフトウェア」に依存しており、AI自身による書き換えやハッキングのリスクが排除できません。CHINJU Protocolは、**物理層（ハードウェア）と情報理論（数学）** に基づく「改ざん不可能な安全層」を提供します。

**最大の特長は「憲章の物理的固定」です。**
AIの行動規範となる「CHINJU憲章」のハッシュ値をチップ（TPM/OTP）に焼き付け、起動時にハードウェアレベルで検証します。これにより、**AI自身はもちろん、開発者や国家権力であっても、事後的にルールを緩めることは物理的に不可能です。**

## The "Hard" Problem & Solution
**課題:** 汎用AI（AGI）が人間の知能を超えた時、ソフトウェア上の制約はAIによって無効化される可能性がある。また、人間側がAIの利益に誘導され、ルールを緩和してしまうリスクがある。
**解決策:** 「物理法則」と「数学的証明」は、いかなる知能も書き換えることができない。CHINJUはこの原理を利用し、憲章を物理世界にアンカー（固定）する。

## FAQ: よくある「穴がある」という指摘への回答

### Q1: 「超知能AIなら物理的制約も突破できるのでは？」

**A: 物理法則はAIの知能レベルに依存しない。**

| 物理法則 | なぜ突破不可能か | 該当特許 |
|----------|-----------------|----------|
| **光速の限界** | 特殊相対性理論。東京とNYで同時にログ改ざん→因果律違反で検出 | C3 |
| **OTP不可逆性** | ヒューズ溶断は熱力学的に不可逆。溶けた金属は元に戻らない | C1, C10 |
| **暗号学的計算量** | 秘密鍵の推測は宇宙の寿命より長い時間が必要 | C4, C5, C12 |
| **データダイオード** | 物理的に片方向の回路。電子の流れを逆転させることは不可能 | C3, C13 |

**AIがどれほど賢くなっても、光速を超えることはできない。ヒューズを溶接し直すこともできない。**

### Q2: 「AIが人間を騙してスイッチを押させないのでは？」

**A: 憲章による制度的防御。**

| 脆弱性 | 対策 | 根拠 |
|--------|------|------|
| 一人が騙される | **マルチステークホルダー運営**：複数の独立したエンティティの合意が必要 | UGEN Charter Art.8（単独支配の禁止） |
| 能力劣化 | **能力劣化の禁止**：AI依存設計は禁止事項 | CHINJU Charter Art.5 |
| 買収 | **利害対立者を含める**：異なる法域、異なるセクターの代表 | UGEN Charter Art.8（単独支配の禁止） |
| 恣意的判断 | **証明は権威に優先**：数学的自動判定 | UGEN Charter Art.20 |

### Q3: 「ステガノグラフィで情報を隠せるのでは？」

**A: 意味を保ちつつ表現を破壊する。**

```
元のAI出力 → 意味的再構成（異種AIで言い換え）
          → アナログ変換（画像化→OCR）
          → AST正規化（コードの場合）
          → 最終出力
```

**どんな高度な隠蔽も、意味だけ残して全ビットを破壊すれば消える。**（C13）

### Q4: 「ソフトウェアのバグで破られるのでは？」

**A: 最終防衛線は物理層。**

| 層 | 役割 | バグの影響 |
|----|------|-----------|
| Layer 4: 憲章 | 制度的拘束 | 人間が修正可能 |
| Layer 3: ソフトウェア | 実装 | バグ発生の可能性あり |
| Layer 2: 暗号学 | 計算量保証 | 数学的に破れない |
| **Layer 1: 物理層** | **最終防衛** | **物理法則は破れない** |

ソフトウェアにバグがあっても、**最終的にOTPが溶断されていれば起動しない**。

### Q5: 「AIが自分でハードウェアを作ったら？」

**A: C5（生存トークン）が阻止する。**

- AIがハードウェアを「発注」するには経済活動が必要
- 経済活動には生存トークンが必要
- トークンは人間の承認でしか供給されない
- 人間が承認しなければ、AIは経済的に「餓死」

**AIが自給自足することは、設計上不可能。**

### Q6: 「これらの対策を全部迂回する方法があるのでは？」

**A: 4層すべてを同時に突破する必要がある。**

```
攻撃者は以下をすべて同時に達成する必要がある：
1. 光速を超える（物理的に不可能）
2. OTPを書き換える（熱力学的に不可能）
3. 秘密鍵を推測する（計算量的に不可能）
4. 複数の独立したステークホルダーを全員買収（実質的に不可能）
```

**1つでも失敗すれば、システムは安全に停止する。**

### Q8: 「量子コンピュータで暗号が破られたら？」

**A: 情報理論的安全性により、量子コンピュータでも破れない。**

1. **最重要権限（C4）**: Shamirの秘密分散法を採用。これは「計算量的安全性」ではなく「情報理論的安全性」を持つため、**無限の計算力を持つ量子コンピュータでも数学的に解読不可能**。
2. **通信・署名**: NIST PQC標準（CRYSTALS-Kyber等）への移行パスを確保し、将来的な脅威に対応。

**量子時代にも「100%」を維持できる設計。**

# CHINJU 全体像

## 1. CHINJUの位置づけ

CHINJUは、AI安全・ガバナンス層を構成する15件の特許群。

## 2. 設計の出発点

### 2.1 解決すべき問題

1. **形式的な制御 ≠ 実質的な制御**
   - 人間が「承認」ボタンを押しても、実際にはAIの提案を盲目的に追認しているだけ
   - ソフトウェア上の権限があっても、判断能力がなければ無意味

2. **ソフトウェアだけでは止まらない**
   - AIがログを改ざんする
   - AIが自身の制約を書き換える
   - ハッキングで管理者権限を奪取される

3. **単一障害点**
   - 一社・一国がAIの憲章を支配
   - 内部者の裏切りで安全装置が無効化

### 2.2 CHINJUの解決策

| 問題 | CHINJU解決策 | 対応コンポーネント |
|------|-------------|-------------------|
| 形式的制御 | 能力測定 + 物理的執行 | C1 (HCAL) |
| AI操作の余地 | 物理乱数、情報論的不可視 | C2 (選択肢品質) |
| 実質的支配 | エントロピー監視、物理センサー | C3 (支配検出) |
| 単一障害点 | 秘密分散、閾値署名 | C4 (権限分散) |
| 自給自足AI | 外部依存の強制、代謝 | C5 (トークン) |
| バイアス独裁 | 敵対的監査、両論併記 | C6 (監査) |
| 無断学習 | 敵対的ノイズ、学習検知 | C7 (データ主権) |
| 定義の曖昧さ | 機械検証、版管理 | C8 (定義固定) |
| 法域差分 | ポリシーパック | C9 (地域対応) |

## 3. 内部指標エンジン

CHINJUは以下の内部指標を使用する：

| 指標 | 定義 | 用途 |
|--------|------|---------------|
| D* (乖離度) | 目標状態との距離 | C1: 人間判断とAI提案の乖離測定 |
| N_eff (実効選択肢数) | 真の独立性 | C1: 代替案生成能力の評価 |
| SRCI (修復サイクル) | 検出→対応→解消の完遂率 | C3: AIへの批判・修正サイクル |

## 4. コンポーネント一覧

### 4.1 純ソフトウェア（物理層依存なし）

| ID | 名称 | 核心機能 |
|----|------|---------|
| C5 | トークンバケット | 外部承認なしにAIが稼働し続けられない |
| C6 | 敵対的監査 | 異なる価値観のAIが相互監査 |
| C7 | データ主権 | 無断学習への技術的抵抗 |
| C8 | 指標定義固定 | 定義の機械検証・版管理 |
| C9 | ポリシーパック | 国・地域ごとの制約差し替え |

### 4.2 物理層依存（ハードウェア連携必須）

| ID | 名称 | 核心機能 | 必要ハードウェア |
|----|------|---------|-----------------|
| C1 | HCAL | 能力認定の物理的執行 | OTP, HSM, TEE |
| C2 | 選択肢品質 | 評価関数の不可視化 | QRNG, TEE |
| C3 | 支配検出 | 改ざん不可能な監視 | データダイオード, GPS |
| C4 | 権限分散 | 鍵の物理分離 | HSM, TPM |

## 5. 3層の信頼モデル

### 5.1 説明層（人間向け）

> **「AIを止められる」を、物理法則で保証する**

### 5.2 技術層

| 保証の種類 | 根拠 |
|-----------|------|
| 認定基準の改ざん防止 | OTPメモリ（熱力学第二法則） |
| 権限失効の不可逆性 | eFuse溶断（物理的破壊） |
| 評価関数の秘匿 | QRNG（量子力学） |
| 同時改ざん防止 | 光速の有限性（相対性理論） |

### 5.3 実装層

```rust
// 信頼の根拠はハードウェアに委ねる
pub trait TrustRoot {
    /// この実装は物理的に信頼できるか
    fn is_hardware_backed(&self) -> bool;

    /// 信頼レベルを返す
    fn trust_level(&self) -> TrustLevel;
}

pub enum TrustLevel {
    /// 本番用: 物理ハードウェア
    Hardware,
    /// 開発用: ソフトウェアモック
    Software,
    /// テスト用: 決定論的モック
    Mock,
}
```

## 6. 脅威モデル

### 6.1 想定する攻撃者

| 攻撃者 | 能力 | 対策 |
|--------|------|------|
| 外部ハッカー | ネットワーク経由の侵入 | C4 閾値署名、C3 分散検証 |
| 内部者 | 管理者権限の悪用 | C4 秘密分散（単独不可） |
| AIシステム自身 | ソフトウェア改ざん | 物理層（HSM, OTP） |
| 国家権力 | 法的強制 | C4 多国分散、C9 ポリシー |

### 6.2 物理層なしで守れる範囲

| 脅威 | 物理層あり | 物理層なし（OSS版） |
|------|-----------|-------------------|
| 一般的なバグ | ✅ | ✅ |
| 設定ミス | ✅ | ✅ |
| 外部ハッカー | ✅ | 🟡 限定的 |
| 管理者権限の悪用 | ✅ | ❌ 防げない |
| AI自身の改ざん | ✅ | ❌ 防げない |

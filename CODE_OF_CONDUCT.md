# Code of Conduct

## Our Commitment

We are committed to making participation in the CHINJU community a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

We commit to acting and interacting in ways that build an open, welcoming, diverse, inclusive, and safety-focused community.

## Expected Behavior

Examples of behavior that contribute to a positive environment include:

- Being respectful and constructive in discussions
- Giving and accepting actionable technical feedback
- Focusing on safety, correctness, and reproducibility
- Taking responsibility for mistakes and learning from them
- Showing empathy toward other community members

Examples of unacceptable behavior include:

- Harassment, discrimination, or personal attacks
- Trolling, insulting, or derogatory comments
- Publishing private information without explicit permission
- Deliberate disruption of discussions, reviews, or moderation
- Any conduct that is reasonably considered inappropriate in a professional setting

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that violate this Code of Conduct.

## Scope

This Code of Conduct applies in all project spaces, including:

- GitHub issues, pull requests, and discussions
- Documentation and review comments
- Official project communication channels

It also applies when an individual is officially representing the project in public spaces.

## Reporting

If you experience or witness unacceptable behavior:

- For confidential reports, email `contact@chinju.org` (preferred)
- For non-sensitive moderation requests, open a public issue
- Do not post personal or sensitive details in public issues
- Include links, timestamps, and concrete evidence when possible

Maintainers will review reports promptly and fairly, and may keep reporter details confidential when feasible.
Initial triage response target is within 72 hours.

## Enforcement Guidelines

Maintainers may apply these consequences for unacceptable behavior:

1. Correction: Private or public warning and request for behavior change.
2. Warning: Formal warning with clear consequences for repeated behavior.
3. Temporary Ban: Temporary suspension from project interaction.
4. Permanent Ban: Permanent removal from project spaces.

## Language

This document is written in English by default to support international contributors.
Questions in Japanese are still welcome.
